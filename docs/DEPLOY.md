# 🚀 部署指南 (Node.js/TypeScript 版)

本系统支持 **Vercel 云端部署**（推荐）和 **本地 Node.js 运行**。

---

## 1. 环境变量配置
无论采用哪种部署方式，都必须配置以下核心变量：

| 变量名 | 必填 | 说明 |
| :--- | :---: | :--- |
| `GEMINI_API_KEY` | 是 | Google Gemini API Key |
| `TIINGO_API_TOKEN` | 是 | Tiingo 股票数据 Token |
| `FRED_API_KEY` | 否 | 美联储宏观数据 Key (不填则无宏观背景) |
| `MONGODB_URI` | 是 | MongoDB Atlas 连接字符串 |
| `TELEGRAM_BOT_TOKEN` | 否 | Telegram Bot Token |
| `JWT_SECRET` | 是 | 自定义随机字符串，用于登录加密 |
| `PASSWORD` | 是 | Web 控制台管理员登录密码 |

---

## 2. Vercel 云端部署 (Serverless)

1. **准备工作**: 将本项目 Fork 或上传到您的 GitHub。
2. **导入项目**: 在 Vercel 控制台点击 `New Project`，导入您的仓库。
3. **配置变量**: 在 Vercel 的 `Settings -> Environment Variables` 中填入上述所有变量。
4. **一键部署**: 点击 `Deploy`。Vercel 会自动执行 `npm run build` 并映射 `api/index.js`。
5. **配置 Cron**: 本项目已内置 `vercel.json` 定时任务。部署后，系统会自动在美东收盘后执行分析。

---

## 3. 本地 Node.js 运行

1. **环境要求**: Node.js v18+ 
2. **安装依赖**:
   ```bash
   npm install
   ```
3. **配置 .env**: 在根目录创建 `.env` 文件并填入变量。
4. **构建并启动**:
   ```bash
   npm run build
   npm start
   ```
5. **访问**: `http://localhost:8080`

---

## 4. 数据库初始化
系统首次启动时，会自动连接 MongoDB。请确保您的 MongoDB Atlas 已开启 `Network Access` (允许所有 IP 或指定 IP 访问)。
